# Contributing to MoodFlix

Welcome! We are excited that you want to contribute to MoodFlix. Please review the following guidelines before submitting your pull requests or logging issues.

## Code of Conduct
All contributors must adhere to our [Code of Conduct](CODE_OF_CONDUCT.md) in all community interactions.

## How Can I Contribute?

### Reporting Bugs
- Search existing issues to verify the bug hasn't been reported.
- Open a new issue containing a clear title, description, steps to reproduce, and the expected vs. actual behavior.

### Suggesting Enhancements
- Check existing issues.
- Explain the suggested enhancement, target benefits, and outline implementation mockups.

### Submitting Pull Requests (PRs)
1. Fork this repository and clone it locally.
2. Create a topic branch from the main branch.
3. Make clean edits, matching the [DEVELOPER_GUIDE.md](DEVELOPER_GUIDE.md) coding patterns.
4. Run all JUnit tests: `.\mvnw test` (or `./mvnw test` on Unix) and make sure they pass.
5. Commit with descriptive messages.
6. Push to your fork and submit a PR to the main branch.
